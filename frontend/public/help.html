Centro de ayuda

MydIAgram convierte lo que escribes en lenguaje natural en diagramas editables (entidad-relación, UML, flujos, arquitectura). Esta guía cubre dos cosas: cómo usar la aplicación y —lo más importante— cómo configurar el modelo de lenguaje que genera los diagramas, ya sea gratis y en local con Ollama o con la API de un proveedor.

¿Con prisa?

Si solo quieres empezar a generar diagramas sin pagar nada, ve directo a Opción A · Ollama en local: instalas un programa, descargas un modelo y listo.

1 Cómo usar la aplicación

La interfaz tiene tres zonas: el lienzo central donde vive el diagrama, el chat a la derecha para conversar con el agente, y la barra de herramientas para editar, exportar y acceder a tu perfil.

Generar un diagrama

Escribe en el campo de texto qué quieres representar y pulsa Enter. Por ejemplo: «una base de datos para una tienda online con clientes, pedidos y productos». El agente clasifica el tipo de diagrama, extrae las entidades y las relaciones, y dibuja el resultado.

Editar el diagrama

Todo lo generado es editable: arrastra los nodos para recolocarlos, haz doble clic sobre un texto para renombrarlo, y usa la barra de herramientas para añadir nodos, cambiar el tipo de relación de una arista o reorganizar el diseño automáticamente.

Refinar con el chat

En lugar de editar a mano, puedes pedir cambios en lenguaje natural: «añade una entidad Factura ligada a Pedido» o «convierte la relación entre Cliente y Pedido en uno-a-muchos». El agente aplica el cambio sobre el diagrama existente.

Historial y papelera

Si inicias sesión con Google, tus diagramas se guardan automáticamente y aparecen en el historial, desde donde puedes reabrirlos. Los diagramas eliminados van primero a una papelera desde la que puedes restaurarlos o borrarlos definitivamente.

Exportar

Desde el menú de exportación puedes descargar el diagrama como imagen PNG o como archivo .mdia (el formato propio, que conserva el modelo de datos para reabrirlo más tarde).

2 Configurar el modelo de lenguaje

¿Qué motor usa MydIAgram?

El agente que genera los diagramas necesita un modelo de lenguaje (LLM) detrás. Tienes dos caminos, y eliges en Perfil ▸ Configuración:

Gratis · privado

A · Ollama en tu ordenador

El modelo se ejecuta en tu propia máquina. No pagas nada y los datos no salen a ningún proveedor externo. Ideal para empezar y para privacidad máxima.

De pago · más potente

B · Proveedor con API key

Usas OpenAI, Anthropic o Google Gemini con tu propia clave. Más calidad y velocidad, pero necesitas una cuenta y pagas por uso al proveedor.

Opción A · Ollama en local (paso a paso)

¿Qué es Ollama? Es un programa gratuito y de código abierto que descarga y ejecuta modelos de lenguaje directamente en tu ordenador, sin conexión a la nube. MydIAgram se comunica con él para generar los diagramas.

Resultado al terminar

Ollama instalado y un par de modelos descargados; en la app eliges el proveedor Ollama (local) y ya puedes generar diagramas gratis.

  1. Descarga e instala Ollama desde ollama.com/download.
    • macOS: descarga el .dmg, ábrelo y arrastra Ollama a Aplicaciones.
    • Windows: descarga e instala el .exe (Windows 10 o superior).
    • Linux: ejecuta en un terminal curl -fsSL https://ollama.com/install.sh | sh.
  2. Comprueba que arrancó. Tras instalarlo, Ollama queda corriendo en segundo plano en http://localhost:11434. Para verificarlo abre un terminal y ejecuta: 
    ollama --version
  3. Descarga los modelos que usa MydIAgram (uno ligero y rápido, otro más capaz). Desde el terminal: 
    # modelo rápido (ligero, ~1.4 GB)
ollama pull qwen3:1.7b

# modelo capaz (más calidad, ~5 GB)
ollama pull qwen3:8b

    Puedes usar otros modelos (por ejemplo llama3.1 o mistral); basta con escribir su nombre en la configuración. Los modelos grandes piden más memoria RAM: si tu equipo es modesto, quédate con el modelo rápido para ambos campos.

  4. Selecciónalo en la app. Abre Perfil ▸ Configuración, elige el proveedor Ollama (local) y escribe los nombres de modelo que descargaste en «Modelo rápido» y «Modelo capaz».

Transporte: «navegador» o «directo»

Con Ollama hay dos formas de que la app llegue a tu modelo local:

ModoCómo funcionaCuándo usarlo
navegador
(por defecto)		 Es tu navegador quien llama a localhost:11434. El cálculo del modelo ocurre en tu máquina; el servidor solo coordina el mensaje. No requiere ninguna configuración de red. La opción normal: tú usas la app desde el mismo ordenador donde corre Ollama.
directo Es el servidor quien llama a Ollama, en la URL que indiques (base_url, por defecto http://localhost:11434). Cuando Ollama corre en otra máquina o en el propio servidor accesible por red.
Privacidad en modo Ollama

La inferencia del modelo ocurre en local y no se envía ninguna API key (Ollama no la necesita). En modo «navegador» el texto del prompt y el resultado sí transitan por el servidor para coordinar la llamada, pero el modelo nunca se ejecuta en la nube.

Opción B · Proveedores con API key

Si quieres más calidad o velocidad, puedes usar un proveedor comercial con tu propia clave. En este modo, la generación ocurre en sus servidores: el prompt y la clave se usan en el servidor de MydIAgram para llamar a la API del proveedor.

Aviso sobre los pasos

Los proveedores cambian sus paneles a menudo: los pasos siguientes son orientativos y pueden no coincidir exactamente con lo que veas. La idea general (crear cuenta → ir a «API keys» → crear clave → copiarla) se mantiene.

OpenAI (claves que empiezan por sk-)

  1. Crea una cuenta en platform.openai.com.
  2. Entra en API keys (menú de tu cuenta) → Create new secret key.
  3. Copia la clave en ese momento (luego no se vuelve a mostrar) y pégala en la app.
  4. Necesitarás añadir saldo o un método de pago en Billing; OpenAI cobra por uso.

Anthropic / Claude (claves sk-ant-)

  1. Crea una cuenta en console.anthropic.com.
  2. Ve a API Keys → Create Key.
  3. Copia la clave y pégala en la app. Configura un método de pago en Billing.

Google Gemini (claves que empiezan por AIza…)

  1. Entra en Google AI Studio · API keys con tu cuenta de Google.
  2. Pulsa Create API key y selecciona o crea un proyecto.
  3. Copia la clave y pégala en la app.
Trata tu API key como una contraseña

Cualquiera con tu clave puede gastar dinero en tu nombre. No la compartas ni la subas a repositorios. Si crees que se ha filtrado, revócala en el panel del proveedor y crea una nueva.

Dos formas de usar tu API key: transitoria o guardada

MydIAgram te deja elegir, de forma deliberada, entre comodidad y minimización de datos:

por sesión · no se guarda por defecto

Clave transitoria
  • La clave vive solo en tu navegador (sessionStorage) y en la memoria de la conexión con el servidor mientras la usas.
  • Nunca toca la base de datos ni el disco: al cerrar la pestaña, desaparece.
  • Ventaja mínima exposición: no queda nada almacenado.
  • Coste tendrás que volver a pegarla cada nueva sesión.

guardada · cifrada

Clave guardada
  • Se almacena cifrada en el servidor (Supabase Vault) y se reutiliza en futuras sesiones.
  • El servidor solo te dice si tienes clave guardada (has_api_key); nunca te la devuelve.
  • Ventaja comodidad: no la vuelves a introducir.
  • Riesgo la credencial reside (cifrada) en el servidor; es un dato más que custodiar. Puedes revocarla cuando quieras desde la configuración, lo que borra el secreto.
Recomendación

Si te preocupa la privacidad, usa la clave transitoria (o directamente Ollama en local, donde no hay clave). Guarda la clave solo si la comodidad te compensa y confías en la custodia cifrada del servidor.

3 Cómo protegemos tus datos

Estos son los criterios de seguridad que sigue la aplicación, descritos tal y como están implementados:

Las API keys

  • Cifrado en reposo. Las claves guardadas se almacenan en Supabase Vault, que las cifra en reposo con una clave maestra custodiada fuera de la base de datos. En la tabla solo se guarda una referencia al secreto, nunca la clave en claro.
  • Nunca vuelven al navegador. La app solo recibe un indicador (has_api_key) de si hay clave guardada. La clave descifrada solo se usa dentro del servidor para llamar al proveedor.
  • Modo transitorio sin persistencia. Si no la guardas, la clave vive solo en memoria de tu navegador y de la conexión, y se descarta al desconectar.
  • Revocable. Puedes borrar la clave guardada en cualquier momento; se elimina el secreto del Vault.

Tu cuenta y tus diagramas

  • Aislamiento por usuario (RLS). Cada fila de la base de datos está protegida por Row Level Security: las políticas de Postgres garantizan que solo puedes leer y escribir tus diagramas y tu configuración. El aislamiento lo impone la base de datos, no solo el servidor.
  • Sesión verificada criptográficamente. El inicio de sesión usa Google a través de Supabase Auth. Cada petición lleva un token firmado que el servidor verifica con las claves públicas (JWKS, ES256) de Supabase — sin compartir secretos.
  • Un único punto de acceso a datos. El navegador nunca habla directamente con la base de datos: toda la persistencia pasa por el servidor (gateway), que transporta tu identidad y deja que la RLS decida.
Transparencia

Para acelerar generaciones repetidas existe una caché global de resultados que no está vinculada a tu cuenta (no contiene tu identificador de usuario). Guarda el prompt y el diagrama resultante de forma compartida y anonimizada entre usuarios.

4 Privacidad y derechos RGPD

MydIAgram trata datos personales mínimos y te da herramientas para ejercer tus derechos del Reglamento General de Protección de Datos directamente desde la app.

Tus derechos y cómo ejercerlos

Acceso y portabilidad (arts. 15 y 20) — recibir una copia de tus datos.

Ve a Perfil ▸ Mis datos y privacidad ▸ Descargar JSON. Obtienes un archivo con tu perfil, tu configuración (sin la API key) y todos tus diagramas, incluidos los de la papelera.

Supresión / derecho al olvido (art. 17) — borrar tu cuenta y todo dato asociado.

En Perfil ▸ Mis datos y privacidad ▸ Eliminar cuenta. Tras confirmar escribiendo ELIMINAR, se borra tu usuario y, en cascada, tus diagramas, tu configuración y tu API key. Es irreversible.

Rectificación y limpieza parcial — sin borrar la cuenta entera.

  • Borrar un diagrama: desde el historial (va a la papelera; luego puedes borrarlo definitivamente).
  • Revocar la API key guardada: en Configuración, sin perder el resto de ajustes.
Minimización de datos

El modelo es «inicia sesión solo para guardar»: puedes usar la app y generar diagramas sin cuenta. Y si eliges Ollama en local con clave transitoria, prácticamente ningún dato personal sensible llega a almacenarse.
